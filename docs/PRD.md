# PRD (제품 요건 정의서)

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 카카오톡 대화분석기 (TalkAnalyse) |
| **목적** | 카카오톡 대화 내보내기 파일(.txt)을 업로드하면 대화 패턴, 감정, 개념설명을 분석해주는 웹 앱 |
| **대상 사용자** | 자신의 카카오톡 대화를 분석해보고 싶은 일반 사용자 |
| **프로젝트 목적** | 포트폴리오 / 기술 학습 |

---

## 주요 기능

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| txt 파일 업로드 | 카카오톡 대화 내보내기 txt 파일을 드래그앤드롭 또는 파일선택으로 업로드 | P0 |
| 대화 파싱 | iOS/Android 포맷 자동 감지, 날짜·발신자·메시지 구조화, 미디어/시스템 메시지 필터링 | P0 |
| 대화 패턴 분석 | 답장 속도, 대화 주도권, 대화 시작 빈도, 시간대별/요일별 활동량, 메시지 길이 통계 | P0 |
| 감정 분석 | Claude API를 활용한 메시지별/참여자별 감정 톤 분석 (긍정/부정/중립), 감정 변화 추이 | P1 |
| 개념 보조설명 | 감정 분석 시 어려운 용어 동시 추출, 대화 뷰에서 밑줄 표시 → 클릭 시 팝오버 설명 (하이브리드 트리거) | P1 |
| 대용량 처리 | ~10MB 전체 분석, ~30MB 배치 분할, ~100MB 샘플링, 100MB+ 업로드 거부 | P1 |
| 분석 대상 | 1:1 대화 + 그룹 채팅 모두 지원 | P0 |
| 팩트체크 | 대화 속 주장의 진위 판별 (웹 검색 연동) | P2 (추후 확장) |

## 동작 방식 (사용자 플로우)

```
1. 사용자가 카카오톡 앱에서 대화 내보내기 (.txt)
2. 웹 앱 접속 → 파일 드래그앤드롭 또는 파일 선택으로 업로드
3. 브라우저에서 txt 파싱 (클라이언트 사이드)
   ├─ iOS/Android 포맷 자동 감지
   ├─ 날짜, 발신자, 메시지, 시간 구조화
   └─ 미디어/시스템 메시지 필터링
4. 대화 패턴 분석 (클라이언트 사이드)
   ├─ 참여자별 메시지 수, 평균 길이
   ├─ 시간대별/요일별 활동 히트맵
   ├─ 답장 속도 분석
   └─ 대화 주도권/시작 빈도
5. 감정 분석 + 개념 추출 (서버 → Claude API)
   ├─ 메시지를 배치 단위로 Claude API 전송
   ├─ 감정 분석 결과 + 어려운 용어 목록 동시 반환
   └─ 대용량 시 배치 분할 / 샘플링 적용
6. 결과 대시보드 표시
   ├─ 대화 패턴: 차트/그래프 (Recharts)
   ├─ 감정 분석: 감정 추이 그래프, 참여자별 감정 비율
   └─ 개념설명: 대화 뷰에서 용어 밑줄 → 클릭 시 팝오버
```

## 기술 스택

| 영역 | 기술 | 선정 이유 |
|------|------|-----------|
| 프레임워크 | Next.js (App Router) | SSR/SSG, 풀스택, 포트폴리오 어필 |
| 언어 | TypeScript | 타입 안정성, 대규모 데이터 구조 관리 |
| 스타일링 | Tailwind CSS | 빠른 UI 구현, 반응형 |
| 차트/시각화 | Recharts | React 친화적, 다양한 차트 지원 |
| AI/감정분석 | Claude API (Anthropic) | 한국어 이해도 높음, 구조화 응답 |
| 파싱 | 자체 파서 (클라이언트) | 카카오톡 txt 포맷 전용, 서버 부하 제거 |
| 배포 | Vercel | Next.js 최적화, 무료 티어 |

## 대용량 파일 처리 전략

| 파일 크기 | 처리 방식 | UX |
|-----------|----------|-----|
| ~10MB (~50,000 메시지) | 전체 분석 | 쾌적한 분석, 진행률 표시 |
| ~30MB (~150,000 메시지) | 배치 분할 처리 | 진행률 + 부분 결과 먼저 렌더링 |
| ~100MB (~500,000 메시지) | 샘플링 적용 안내 후 처리 | 샘플링 동의 팝업 → 시간대별 균등 샘플링 |
| 100MB+ | 업로드 거부 | 기간 나눠서 내보내기 안내 |

### 처리 아키텍처

```
[브라우저]                              [서버 (Next.js API)]
  │                                       │
  ├─ txt 파싱 (FileReader)                │
  ├─ 구조화 JSON 생성                     │
  ├─ 대화 패턴 분석 (로컬)               │
  │                                       │
  └─ 구조화 데이터 전송 ──────────────→  ├─ 배치 분할
     (원본 txt 아님, JSON)                ├─ Claude API 호출 (감정 + 용어)
                                          └─ 결과 병합 후 응답
```

## 카카오톡 txt 파싱 규격

### 지원 포맷

```
# 공통 날짜 구분선
--------------- 2024년 3월 15일 금요일 ---------------

# Android 포맷
[홍길동] [오후 2:30] 오늘 회의 몇 시야?

# iOS 포맷
2024. 3. 15. 오후 2:30, 홍길동 : 오늘 회의 몇 시야?
```

### 파싱 시 처리 항목

| 항목 | 처리 방식 |
|------|----------|
| 날짜 구분선 | `--- YYYY년 M월 D일 ---` 패턴 매칭 |
| 메시지 | `[이름] [시간] 내용` (Android) / `날짜, 이름 : 내용` (iOS) |
| 미디어 | `[사진]`, `[동영상]`, `[파일]` 등 → 별도 카운트, 분석 제외 |
| 시스템 메시지 | 입장/퇴장/강퇴 알림 → 필터링 |
| 멀티라인 | 줄바꿈 포함 메시지 → 다음 메시지 패턴 전까지 이어붙이기 |
| 삭제된 메시지 | "삭제된 메시지입니다" → 별도 카운트 |

## 제약 사항

1. **개인정보**: 업로드된 대화 파일은 서버에 저장하지 않음. 분석 완료 후 즉시 폐기
2. **AI 비용**: Claude API 호출 비용 발생. 배치 분할/샘플링으로 비용 관리
3. **정확도**: 감정 분석 및 개념설명은 AI 기반이므로 100% 정확도 보장 불가. UI에 "AI 참고 의견" 명시
4. **포맷 의존성**: 카카오톡 내보내기 포맷 변경 시 파서 업데이트 필요
5. **Vercel 무료 티어**: Serverless 함수 10초 타임아웃 → 배치 분할로 대응
